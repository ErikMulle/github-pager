## 🔒 Práticas de Segurança e Governança Implementadas (PortfolioHUB)

Em conformidade com os requisitos de segurança e gestão estabelecidos para o desafio, foram aplicadas as seguintes políticas de proteção no ambiente do GitHub[cite: 1]:

*   **Ocultação de Dados Sensíveis:** Utilização estrita do arquivo `.gitignore` para impedir o vazamento acidental de chaves de API, credenciais ou arquivos locais de configuração do ambiente[cite: 1].
*   **Gestão de Credenciais Seguras:** Configuração para armazenamento de tokens de autenticação e variáveis sensíveis via *GitHub Actions Secrets*, evitando a exposição dessas informações no código-fonte público[cite: 1].
*   **Controle de Acesso e Revisão:** Proteção da branch principal (`main`) configurada para exigir *Pull Requests* e revisão antes de qualquer integração, impedindo modificações diretas sem validação[cite: 1].
*   **Mitigação de Riscos via IA:** Uso do Google GEMINI como guia para auditoria interna de boas práticas de código, arquitetura e segurança da informação durante a implantação[cite: 1].
